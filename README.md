
# Hokkai Gakuen News 1 - 管理用サイト (CMS)

こちら管理用サイトです。このアプリケーションは、北海学園大学一部新聞会の公式公開サイト（表示用サイト）を遠隔制御するための専用管理システム（管制塔）です。

## 連携体制 (Ecosystem & Command Chain)
このプロジェクトは以下の4つの役割が厳格な指揮系統に基づいて連携し、運用されます。

1. **作成者 (Creator/User)**: 
   - 全体の動向を監視し、各ロール間の発言を伝令します。
   - 違和感がある際の提案や、根本的な修正依頼を統括する最高責任者です。

2. **管制 (Command/External Gemini)**: 
   - システム全体のロジック監修者です。
   - 表示用・管理用それぞれのサイトから報告されるエラーに基づき、具体的な修正指示を出します。
   - サイト間（管理用⇔表示用）の重要な伝令・連絡のハブとなります。

3. **管理用サイト (本アプリ/AI)**: 
   - Firestoreへのデータ供給、表示サイトのステータス制御を行う**実務実行部隊（バックエンド）**です。
   - 管制からの指示に基づき、自身のコードやセキュリティルールを修正します。
   - 表示用への伝言は必ず管制を通します。

4. **表示用サイト (Front-end/AI)**: 
   - Firestoreのデータをリアルタイムに表現する**公開画面（フロントエンド）**です。
   - 管制からの指示に基づき、表示ロジックやフィルタリングを修正します。
   - 管理用が供給するデータの仕様（下記）を厳守して描画します。

## 緊急伝令プロトコル (Emergency Protocol)
表示用サイト側でデータの不整合やエラー（インデックス作成待ちなど）が発生した際、表示用サイトの AI が自己のバグと誤認して不必要な修正を行わないよう、管理用サイトは以下の設定ドキュメントを通じて「状況」を送信します。

- **通信パス**: `/settings/maintenance`
- **フィールド**: `systemStatusMessage` (string)
- **表示用サイトの挙動**: このメッセージが空でない場合、画面上部などにステータスバーとして表示し、管制からの指示を待機すること。

## 表示用サイトへのデータ提供仕様 (Data Contract)
表示用サイトは、管理用サイトがFirestoreに書き込む以下の仕様に基づいてクエリを作成してください。

### 1. 記事表示 (Articles)
- **パス**: `/articles/{articleId}`
- **フィルタ**: 必ず `isPublished == true` の条件で取得すること。
- **ソート**: `publishDate` (降順) または `updatedAt` (降順) を推奨。
- **フィールド名注意**: `isPublished` (boolean), `articleType` ("Standard" or "Note")。

### 2. メンテナンス・ステータス制御 (Maintenance)
- **パス**: `/settings/maintenance`
- **フィールド**: 
  - `isMaintenanceMode`: `true` の場合、全ページをメンテナンス画面へ切り替えること。
  - `systemStatusMessage`: 現在のシステム状況（例：インデックス作成中）を表示すること。
